Hi all,

I have just started to use the UCT-IMS client for testing OpenIMSCore. First
of all I am very happy about the initiative to offer an Open Source IMS
client. An important missing part is going to be closed now.

But one of the first observations I made is, that the implementation of
PRACK is not correct. According to RFC 3262 the UAS who wants to send a
provisional response reliably MUST include a Require header field containing
the option tag 100rel and MUST include a RSeq header field. Both header
fields are missing in the provisional response, but the UAC sends a PRACK
request anyway. This PRACK request cannot be matched to any provisional
response without RAck header field, so it is useless.

I hope my comment is helpful. Maybe the problem is already known and the
implementation of the PRACK mechanism is still to be finalised. 

cheers
Franz
