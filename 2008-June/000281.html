Hi,

When you set up a session the bandwidth requirements are included in the 
SIP signaling (e.g. voice PCMA codec specifies 64KB).  This is converted 
to Bytes when passed down the Rx interface to the PCRF. (e.g. voice PCMA 
codec - 64KB = 65536B)

The domain policies specify a maximum uplink and downlink bandwidth for 
each domain - this is an arbitrary amount and can be set in the 
policies/domain_policies.xml file (It is set to 20000KB = 20480000B for 
open-ims.test).  When a session is allowed and enforced in this domain 
its allocated bandwidth is removed from the total amount - if you 
examine the individual policy rules for each session in the Web 
interface you can see the allocated bandwidths for each session and for 
each associated IP Flow.  When the session is terminated, and the 
transport layer is reconfigured accordingly the bandwidth is returned to 
that domain (After terminating all sessions - the domain bandwidth 
should return to the original amount of 20480000B for both uplink and 
downlink).
