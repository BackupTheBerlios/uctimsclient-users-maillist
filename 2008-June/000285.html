Hi,

The specified Uplink and Downlink Bandwidths for each domain are 
arbitrary amounts set in the domain policies.  In a practical 
environment this information would be gathered by QoS reporting and 
Resource Monitoring mechanisms where the network topology is discovered 
and reported upon (Or hard coded in.)

A session should only be rejected because of bandwidth restraints if the 
Uplink or Downlink bandwidth for that domain cannot handle the requested 
amount.  The codec used for a session determines its bandwidth 
requirements.  I'm not sure why you getting sessions rejected - check 
the dynamic policies through the web interface.  Perhaps you have a lot 
of sessions that were not properly cancelled/terminated and hence 
haven't been removed. You can check the available bandwidth for each 
domain through the web interface as well - there is also an option to 
set all policies back to default (To remove old sessions).

Regards,
Richard.
