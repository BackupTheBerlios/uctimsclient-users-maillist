Hi All,

While testing the presence service using XCAP, I am stuck up some issues. 
Following are the details:

For OpenSER 1.3.x, I pulled out Revision 2327 branch using SVN through 
link: http://openser.svn.sourceforge.net/svnroot/openser/trunk/
Platform: PCLinux

The build is successfull using this command: make prefix=/ 
include_modules="mysql presence presence_xml mi_fifo mi_xmlrpc" install . 
The executables, .cfg and modules are installed at the path mentioned in 
http://uctimsclient.berlios.de/xcap_server_howto.html.

/sbin - openserunix; openser_textdb.sh; openser; openserctl; 
openser_mysql.sh

/etc/openser - openserrctlrc; openser.cfg

/share/openser/dbtext - openser_db

/lib/openser - libraries

I have no earlier versions of OpenSER running on my machine, cleared all 
previous files and started fresh build.

ISSUES:

I am using sample cfg file given at 
http://uctimsclient.berlios.de/openser.cfg.xcap

1. As mentioned in the link: 
http://openser.org/dokuwiki/doku.php/install:openser-from-svn, following 
scripts should be installed.
openser_mysql.sh - script to create and manage the MySQL database (for 
version < = 1.2.X) 
openserdbctl - script to create and manage the Databases (for version > 
1.2.X) 

I am working on version 1.3.X(Revision 2327), but I am not able to find 
"openserdbctl" script to create Databases.


2. While running the command - "openser_mysql.sh create" or 
"./openser_msql.sh, directory - /sbin
I am getting the error: ": bad interpreter: No such file or directory/sh"

(Note: Appropriate call to an interpreter on the first line (ie #!/bin/sh) 
is there)

3. While running the command - "openser start", directory - /etc/init.d
I am getting the error: 

O(30201) parse error (1,1-2): syntax error
O(30201) ERROR: cfg. parser: unexpected EOF in unclosed string
ERROR: bad config file (1 errors)

(Note: According to Openser documentation, this error is thrown from file 
"lex.yy.c)


Please give me the pointers to resolve the above mentioned issues asap... 

Thanks in advance.
