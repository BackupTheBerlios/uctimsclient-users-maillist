Hello all,

I'm doing some basic tests with regular SIP phones (neither preconditions 
nor P-service-routers) and the RTPProxy active on the P-CSCF-PDF. Both 
registered on the same P-CSCF.

Some strange things when calling A -> B:

Let the media ports be these:  A:pa <-> R:pr1 <-> R:pr2 <-> B:pb

- PDF and PEP do not support ports and bandwidth.

- When the 200 OK is passed thru all the components, three interactions 
are made with PDF:

        (1) Allow: R:pr1 --> A:pa
        (2) Deny: R:pa --> A:pr1 (wrong!)
        (3) Allow: R:pr1 --> A:pa

All are done with the same 'Session ID' so, as ports are not supported, 
number (2) is the same as number (1) so it is a deny.

- The only direction reserved is: from P-CSCF to caller. Why?

- Hanging up:

        (4) Deny: R:pa --> A:pr1 (wrong!)

But it works because is the same session and ports are not considered.

- EF MANGLE entries are not deleted when session ends

- The PDF does not end clean typing 'exit'

Is this behaviour usual?

Pedro
