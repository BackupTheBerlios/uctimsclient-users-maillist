Hi all

The client released last week Wednesday still contained the registration 
bug, which meant the client crashed if anything but AKAv1 authentication 
was used.

I've corrected the offending code and uploaded it to Berlios. I have not 
bothered to change the version number as it is a relatively small fix.

The client should now happily register with AKAv1, AKAv2 and plain MD5. 
Please note that you must change the authentication scheme for each user 
in the FHoSS and the scscf.cfg file, otherwise you will get the error: 
"HSS returned no authentication vectors"

Hope this helps. Let me know if anything else is still broken :-)

Regards

Dave
