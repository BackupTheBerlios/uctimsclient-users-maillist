Hello all,

 

I am running UCT IMS Client with the UCT IPTV indirection AS plus VLC as streaming server. I am testing the QoS policy with a DIAMETER server connected to the P-CSCF of the Open IMS Core. I have already activated the Rx interface and I just wanted to know how the P-CSCF issues the AAR messages during the VoD session establishment.

 

However, when I start a VoD session from UCT IMS Client against the UCT IPTV Indirection AS, the INVITE's SDP includes the Multimedia information related to the Audio codecs configured in "Preferences". I think this should not be that way. It should include and SDP with the bandwidth necessary for the RTSP session that will be established.

 

Furthermore, when RTSP session is established with a PLAY commando, the UCT IMS Client doesn't send any SIP INVITE + SPD towards the P-CSCF in order to reserve bandwidth for video and audio traffic. According to the standard this second SIP Session should be started from UCT IM Client before issuing the PLAY command (once all resources has been reserved).

 

Does anybody know it this is or will be implemented?

 

 

Thanks a lot

 

Victor
