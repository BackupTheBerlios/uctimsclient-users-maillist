>     Richard Spiers wrote:
>>     Hi Melbourne
>>
>>     By default, the client uses v4lsrc and /dev/video0. You can
>>     change this behavior by looking in the source code (most likely
>>     gstreamer.c) and changing the v4lsrc to v4l2src, i.e. something
>>     like this
>>
>>     camera_src = gst_element_factory_make("v4l2src", "camera_src");
>>
>>      and by passing the correct device name as follows:
>>
>>     g_object_set (G_OBJECT (camera_src), "device","/dev/video1", NULL);
>>
>>     (where camera_src is the v4lsrc or v4l2src element that gets created)
>>
>>     Let me know if that sorts you out.
>>
>>     Also please be aware that recent changes in the linux kernel have
>>     created some webcam problems. Some functioning webcams no longer
>>     work - the issue has reportedly been fixed but regressed again in
>>     a later version of some modules.
>>
>>     Something to bear in mind if you think about upgrading to
>>     intrepid, although it should have been fixed by now.
>>
>>     Kind Regards
>>     Richard Spiers
