> Hi,
>
> Without any wireshark trace it is tricky to debug.  Though I do remember
> an issue with compatability between the branch and trunk code packages.
>
> See:
> http://lists.berlios.de/pipermail/openimscore-cscf/2008-August/001881.html
>
> The more up to date trunk code introduced support for a qop parameter,
> but the older branch code does not yet support qop. So if you use the
> trunk code for  the I-CSCF and S-CSCF, but the branch code for the
> P-CSCF, the P-CSCF incorrectly parses the new parameter and the client
> crashes.
>
> You should just be able to disable the qop parameter in scscf.cfg for
> this to work.   Alternatively you can incorporate qop support in the
> branch as suggested in the earlier thread :-) .
>
> If this does not solve the problem please send a wireshark trace for
> debugging.
>
> Richard.
