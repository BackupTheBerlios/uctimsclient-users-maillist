Hi,

The output at the P-CSCF is not really an error, but just debugging 
output.  The PCRF is not running and hence the Diameter session can not 
be created.

 I have had an issue when using the ims.qos.sh script where the HSS for 
some reason doesn't start.  If you just run each script separately in a 
new tab it works. 

I am not sure why your I and S-CSCF are crashing.  Are you running the 
latest SVN version of the code? Does the P/I/S CSCF and HSS work 
correctly when using the OSIMS trunk version (i.e. with the policy control)?
