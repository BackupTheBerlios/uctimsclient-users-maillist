Hi Indranil,

This exception is just another synchronization issue with diameter peers 
starting up at different times; if it does not persist then it should be 
OK.   Both these errors should be changed to warnings in the next release.

For testing and checking that the interfaces are working correctly see 
/*Step 7: Testing*/ of the HOW TO: 
http://uctimsclient.berlios.de/policy_control_howto.html 
<http://uctimsclient.berlios.de/policy_control_howto.html>

Also have a look at 
http://uctimsclient.berlios.de/policy_control_scenario.html for some 
simple testing scenarios.

Regards,
Richard.
