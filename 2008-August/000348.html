Hi Zerin,

Out of interest - which presence server are you using?

You can run your presence server with no authentication where anyone can 
subscribe to anyones presentity - in this case anyone who has added 
alice to their buddy list, is a watcher of alice.

Another option is to enable authentication - now whenever someone 
subscribes to alice's presentity she must authorize them before they can 
become a watcher. In the UCT IMS Client the watchers tab is provided for 
this scenario.

For this you need interaction between your presence server and an XCAP 
server.  We use OpenXCAP (http://www.openxcap.org/) and the interface 
between OpenSER and OpenXCAP uses MySQL.  We used to have a HOW TO on 
the website and provided support for this, but we found that OpenSER and 
XCAP were changing too fast and we struggled to keep our documents up to 
date with the latest releases - so for this you will need to look at the 
OpenSIPS and OpenXCAP websites.

The XDMS Server Tab is just a way to test the correct functionality of 
your XCAP server - it is separate from the presence functionality.  It 
just allows you to upload and download XML docs to and from your 
server.  It is useful for debugging when first setting up your server.  
You need to set up your XCAP username, password, address, etc. in the 
XCAP preferences for this to work correctly.

> Hi Zerin,
>
> With only this information it is difficult to diagnose your problem - however here are the basic steps you need to follow to use the presence service.
>
> 1.    You need to have a presence server enabled (We use OpenSER with 
> its presence module - I notice that OpenSER has become OpenSIPS. see http://www.opensips.org/index.php?n=Resources.DocsPapPa for presence server info.)
>
> 2.    You need to configure iFC in the FHoSS web interface to forward 
> all presence related (SUBSCRIBE and PUBLISH messages with event -
> presence) to the address and port of your running presence server.
>
> 3.    Then you can register Alice and Bob (both with presence enabled) 
> and publish their status' under the presence tab.
>
> 4.    Add bob as Alice's buddy (and vice versa) and note that when new 
> status' are published, buddies are notified.
>
> If you have followed all these steps and still have problems, post a full wireshark trace.
>
> Regards,
> Richard.
