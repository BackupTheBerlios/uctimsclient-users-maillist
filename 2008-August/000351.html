Hi,

This error occurs if the client detects an audio component (m=audio 
line) in the SDP body of the SIP messages and the specified codecs in 
the SDP body do not match any supported by the client.

Are you using two UCT IMS Clients or are these interoperability tests 
involving another client?  The UCT IMS Client supports GSM, a-Law, u-Law 
and MP2 codecs (most common audio codecs), so if another client requests 
a different codec this error will be produced.  Also both clients should 
be the same version (preferably the latest 1.0.12) as further codec 
support has been added in later releases.

Regards,
Richard.
