Hi Miroslave

ETSI TISPAN specifications define the use of SIP and RTSP methods for 
interaction between the client, the Application Server and the Media 
Control Function (ETSI TS 183 063 to be exact).

At present the solution does not conform to these ETSI standards however 
we are looking at implementing a fully standards compliant solution but 
I can't give you a ETA on when this will be complete.

Regards

> > The so called trick play functionality (PAUSE, FAST FORWARD, etc) are
> > handled by the RTSP protocol and not SIP. The INVITE and BYE messaging
> > is used to speak to the application server and for the invite obtains
> > the RTSP address of the media the client is requesting. Thereafter a
> > RTSP session is established between the client and the media server.
> >
> > You can look in rtsp.c but you won't find any specific rtsp commands,
> > rather the high level commands that libvlc provides in order to control
> > the media stream.
