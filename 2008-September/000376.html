Hi Agung

Yes this is a bug with the current version of the client and only 
pointed out to us a short time ago. Somewhere along the line the support 
for updating of the preferences while the client is still running was 
removed. It saves the preferences to a file on exit so when the client 
is run again the preferences are all correct but having to restart the 
client after each preference change is far from ideal.

Thank you for pointing this out and we will look into fixes this issue 
with the next release.
