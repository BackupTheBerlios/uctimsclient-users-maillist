Hi Khaldon,

This first implementation does not strictly follow IPtv standards - the 
content indirection implementation with the RTSP address attached to the 
200 OK message is our own implementation. But does not violate any 
IMS/SIP/RTSP functionality.

However  after this first attempt - we should shortly have a more 
standards based implementation (based on ETSI TISPAN specs which define 
two methods for combining RTSP and SIP functionality.) 

We will post more info on this once the prototype is ready for release.

Regards,
Richard.
