Hi,

The cause of the crashes in UCT IMS Client v1.0.10 has been identified.  
The gStreamer libraries moved RTP packet validation, and this caused 
incompatibility issues when using the UCT IMS Client NAT traversal 
methods (The small packet sent to open the NAT from the inside was 
identified as an RTP packet and corrupted the gStreamer pipes.)

Until this issue has been resolved with the gStreamer libraries the NAT 
Traversal method has been removed. UCT IMS Client v 1.0.11 has been 
released to address this issue.  There are also general bug fixes, 
memory management fixes, and MP2 audio codec support.

Regards,
Richard.
