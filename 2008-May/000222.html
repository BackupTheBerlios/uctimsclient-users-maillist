Hi All,

This is just a quick update on the status of the UCT IMS Client and 
related projects.

The SIP stack on which the UCT IMS Client is based - the eXoSIP/oSIP 
stack - has been updated and there have been some slight changes to the 
API.  Hence all UCT IMS software that uses this stack has been updated 
to be compatible with the latest libraries. To use the latest versions 
of the UCT IMS Client (v 1.0.10)  and the UCT IPTv Server (v 1.0.2) you 
will need to update the eXoSIP and oSIP libraries to the latest version 
(libexosip2 v3.0.3 and libosip2 v3.0.3) - these packages are in the 
standard Ubuntu Hardy (8.0.4) repositories.

Additions to the latest UCT IMS Client include general bug fixes, Ubuntu 
Hardy compatibility, improved XDMS error detection and 302 re-direction 
support.

The UCT Policy Control Framework has also been updated - additions 
include general bug fixes, standardization conformance, policy overhaul 
and improved policy enforcement and RTP support.

As always, send any bugs, feature requests or comments to 
uctimsclient-users at lists.berlios.de and find more information on the 
website http://uctimsclient.berlios.de.

Regards,
Richard.
