We are testing the Segmented QoS signalling with the UCT IMS Client 1.0.11 
and we see some issues:

Messages analysis between two UCT IMS Clients:

INVITE: does not show 'Allowed' methods in SIP.
183 Session Progress: does not show 'Allowed' methods in SIP. It should 
not wear a "requiere:precondition" (it is an answer).
PRACK (for 183SP): strange that it requests "a=qos:conf remote sendrecv"
200 OK (for PRACK): it should carry SDP data.
UPDATE: strange that it requests "a=qos:conf remote sendrecv".
200 OK (for UPDATE): the "require: pre-condition" is wrong typed and can 
be omited (it is an answer). It requests "a=qos:conf remote sendrecv" but 
it is already confirmed!!!
180 RINGING: It should not carry SDP.
PRACK (for 180R): it should not transport SDP.

I'm following "The 3G IP multimedia Subsystem (IMS)" book.
