Hi

The ability to register the same IMPU from many different IP addresses is
not a bug. It is in fact one of the greatest features of IMS. The server
will fork your calls to both clients and you can choose which one to answer.


I believe the GRUU feature of SIP will solve this problem for you, but as
far as I recall it was never fully implemented in the client and I'm not
sure if the FOKUS core supports it either.

Regards

David
