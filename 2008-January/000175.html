Hi Abhinav

I'm glad it's working. I'm not sure about the error message - it might 
have something to do with the audio codecs. I wouldn't worry too much 
about it.

I think I know why it's crashing though. I commented out the code for 
receiving regular video calls so that you could use the ximagesink for Iptv.

But the client is still trying to close this pipeline when the call 
ends. If you comment out all references to

destroyVideoRxPipeline(ca);

in the common_exosip_event_handler it will probably stop the 
segmentation faults. Just a guess though.

Regards

Dave
