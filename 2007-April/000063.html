Ramachandran solved the problem like this:


Actually if you configure P-CSCF address in UCTIMSClient as IP address, ACK
is a problem, which further leads to not initializing the RTP ports. In this
case, one way RTP packets are sent.

 

However if you configure P-CSCF address as domain in UCTIMSClient, for
example "pcscf.open-ims.test:4060", ACK is sent properly and hence there is
a two way RTP.



It seems that the pcscf handles ACK differently and this causes a problem.

- Dave
