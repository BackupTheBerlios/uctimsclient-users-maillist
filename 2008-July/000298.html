Richard is currently integrating in some rtsp client support for the UCT
IMS client which will allow you request a stream via rtsp and play it. I
have created a server application that at present returns an rtsp
address to any client that requests an IPTV channel or video on demand.

At present we are working on adding hss lookup authentication for
clients requesting streams from the IPTV application server and then
linking that in with a media control function which will be responsible
for interfacing between the client and the media server (Ensures
security and will allow the use of any media server you wish, eg
Videolan, Darwin, etc since it won't need to speak SIP).
