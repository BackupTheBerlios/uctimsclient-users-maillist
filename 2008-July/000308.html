Hi.

I tested with v1.0.11 and still have no problem adding/removing buddies 
and receiving status notifications.

I have looked at your presence config file and cannot see anything 
obviously wrong.  However attached is the config file we use, it is much 
simpler and ignores all messages except PUBLISH and SUBSCRIBE.  Try with 
this config file and see if you have any luck.

Also ensure that your trigger points are correctly set in the FHoSS.  
I.e. only PUBLISH and SUBSCRIBE messages with "presence" in the SIP 
header are forwarded to your presence server.

If you still have problems please include a wireshark trace from the 
machine running the core and presence server.

Regards,
Richard.
