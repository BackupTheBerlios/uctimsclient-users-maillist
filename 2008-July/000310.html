Hi Prateek,

The MSRP support is not completely integrated yet and is still 
experimental - currently it only supports session based IM over MSRP.

If you enable session IM under the preferences and initiate a session 
(using the normal call button) you will see that the MSRP info is 
included in the SDP body of the messages and an IM window is opened upon 
call establishment - messaging is then sent over MSRP.

The Instant Messaging tab still only uses Pager Mode IM with the SIP 
MESSAGE method used for transport - I should probably include a note 
somewhere that this is the case. It introduced all kinds of 
complications integrating it here as well, but this will be done in the 
next release.

Regards,
Richard.
