Hi Emina,

Your 3G network likely assigns you a private address for security and 
privacy reasons.  So without an RTP proxy two-way media between one 
client on HSDPA (private IP) and another client on a public IP will not 
be possible.  Though you should get one-way media flowing from the 
client behind the NAT to the client with the public IP. 

Perhaps your LAN and WLAN also assigns private IPs, but all are in the 
same subnet.  If both clients are connected via HSDPA (same 3G network) 
do you get any media flowing? If they are both private IPs but in the 
same subnet it should work.

If this is the problem, most 3G networks allow access to an Open APN 
which assigns public IPs.  This is needed for applications like remote 
desktop login.  You will have to contact your provider about this.

Richard.
